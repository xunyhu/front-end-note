# 面试题：如何做错误监控？（标准回答）

在前端项目中，错误监控的核心目标是：**及时发现问题、定位问题、还原问题，并减少线上事故影响范围**。

---

## 一、错误类型划分

前端错误一般分为三类：

### 1. JS 运行时错误

* 代码异常（undefined、null 报错）
* 组件渲染错误
* Promise 未捕获异常

---

### 2. 资源加载错误

* JS/CSS 文件加载失败
* 图片资源加载失败
* CDN 异常

---

### 3. 接口错误

* 请求失败（网络错误）
* 后端返回异常 code
* 超时

---

## 二、常见监控方式

### 1. window.onerror

用于捕获同步 JS 错误：

```js
window.onerror = function (message, source, lineno, colno, error) {
  console.log('JS错误:', message)
}
```

---

### 2. window.addEventListener('error')

可以捕获资源加载错误：

```js
window.addEventListener('error', (event) => {
  console.log('资源错误:', event.target)
}, true)
```

---

### 3. unhandledrejection

捕获 Promise 未处理错误：

```js
window.addEventListener('unhandledrejection', (event) => {
  console.log('Promise错误:', event.reason)
})
```

---

## 三、接口错误监控

在 request 层统一处理：

* 在 axios response interceptor 中捕获错误
* 记录 status code / message / request 参数

---

## 四、错误上报机制

通常会将错误统一上报到日志系统：

* Sentry
* 自研日志平台
* 埋点系统

上报内容包括：

* 错误信息
* 页面路径
* 用户信息（userId）
* 设备信息
* 请求参数

---

## 五、用户行为关联（高级）

为了更好定位问题，会记录：

* 用户操作路径（点击链路）
* 页面停留时间
* 最近请求记录（request log）

用于“错误复现”。

---

## 六、小程序中的错误监控

在小程序（Taro）中：

* 使用 `App.onError` 捕获全局错误
* 使用 `onUnhandledRejection` 捕获 Promise 错误
* 使用 request wrapper 统一上报接口错误

---

## 七、实际项目中的做法

在我的项目中：

* 封装了 request 层统一捕获接口错误
* 使用全局 error handler 处理 JS 异常
* 接入日志上报（类似 Sentry 思路）
* 对关键链路（下单、支付）增加额外埋点

---

## 八、总结

错误监控的核心是：

> **全局捕获 + 统一上报 + 可追踪用户行为 + 快速定位问题**

目标是让线上问题从“用户反馈”变成“系统主动发现”。
