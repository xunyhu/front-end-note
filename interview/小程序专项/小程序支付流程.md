# 面试题：小程序支付流程？（标准回答）

在餐饮/零售类小程序中，支付是核心链路之一，整体流程可以分为：**下单 → 获取支付参数 → 调起支付 → 支付结果处理**。

---

### 一、下单阶段

用户在结算页点击“去支付”后，前端会先调用后端的下单接口，生成订单。

* 后端会进行库存校验、价格校验等
* 返回订单号以及后续支付所需的标识

---

### 二、获取支付参数

前端拿到订单号后，再调用后端的“统一下单接口”，由后端去调用微信支付接口，生成支付参数。

后端返回给前端的数据一般包括：

* timeStamp
* nonceStr
* package
* signType
* paySign

---

### 三、调起微信支付

前端通过小程序 API 调起支付：

```js
wx.requestPayment({
  timeStamp,
  nonceStr,
  package,
  signType,
  paySign,
  success: () => {},
  fail: () => {}
})
```

用户在微信支付弹窗中完成支付。

---

### 四、支付结果处理

支付结果不能完全依赖前端回调，一般需要结合后端通知来处理：

* 前端 success：提示“支付成功”，并跳转订单页
* 后端支付回调（notify）：更新订单状态为已支付

为了保证数据一致性，通常会做：

* 支付成功后主动查询订单状态
* 或者通过轮询方式确认支付结果

---

### 五、异常处理

需要考虑以下情况：

* 用户取消支付
* 支付失败
* 网络异常

前端一般会给出提示，并允许用户重新发起支付。

---

### 六、总结

小程序支付流程的核心是：

> **前端负责调起支付，后端负责订单和支付结果的最终确认。**

在实际项目中，要重点保证：

* 订单状态的准确性（以后端为准）
* 防止重复支付
