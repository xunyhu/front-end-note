# 面试题：小程序和 H5 的区别？（标准回答）

小程序和 H5 在本质上是两种不同的运行环境和开发体系，主要区别可以从运行机制、性能、能力以及开发方式几个方面来看。

小程序没有 DOM/BOM，逻辑层与渲染层分离（双线程），事件通过 setData 触发渲染

1. 运行环境

    * **H5**：运行在浏览器中，依赖 WebView
    * **小程序**：运行在微信环境中，逻辑层和渲染层分离（JSCore + WebView）

    小程序不是一个标准浏览器环境，有自己的一套运行机制和 API。

2. 性能差异

    * 小程序：

        * 由于逻辑层和渲染层分离，通信有开销，但整体性能更稳定
        * 启动速度较快（有缓存机制）

    * H5：

        * 完全依赖浏览器性能
        * 首屏加载容易受网络影响

    整体上，小程序在弱网和复杂业务场景下体验更稳定。

3. API 能力

    * 小程序：

        * 提供丰富的原生能力（如支付、登录、定位、扫码等）
        * 通过 `wx.*` API 调用

    * H5：

        * 依赖浏览器 API
        * 一些能力需要通过第三方 SDK（如 JS-SDK）实现

4. 路由机制

    * 小程序：

        * 基于栈的路由（navigateTo、redirectTo 等）
        * 页面数量有限制

    * H5：

        * 基于 History API
        * 更灵活

5. 开发方式

    * H5：

        * 使用标准前端技术（HTML、CSS、JS）
        * 框架如 React、Vue

    * 小程序：

        * 有自己的一套组件和语法体系
        * 可以通过 Taro 等框架使用 React 开发

6. 发布与更新

    * 小程序：

        * 需要审核发布
        * 有版本管理

    * H5：

        * 部署即生效
        * 更新灵活

7. 小程序的 setData 与 React 的 setState 有何异同

    setData 会序列化数据并通过 Native 传递，频繁调用性能差；React 的 setState 批量更新（V18 自动批处理）。

8. 如何优化小程序首屏加载速度

    分包加载、预请求、onLoad 阶段先渲染骨架屏、避免 setData 传输大数据。

## 总结

小程序和 H5 的核心区别可以总结为：

小程序是“类原生应用”，强调性能和能力；H5 是“浏览器应用”，强调灵活和开放。

在实际项目中，会根据业务场景选择，比如：

* 高转化、强支付场景 → 小程序
* 内容展示、活动页 → H5
