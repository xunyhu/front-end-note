# 面试题：如何做小程序登录？（标准回答）

小程序登录的核心是：**通过微信提供的临时登录凭证 code，换取后端的用户标识与登录态（token）**。

整体流程可以分为：**获取 code → 后端换取 openid/session_key → 生成登录态 → 前端保存 token**。

---

### 一、获取登录凭证 code

小程序端通过 `wx.login` 获取临时登录凭证 code：

```js
wx.login({
  success: (res) => {
    const { code } = res
  }
})
```

这个 code 是一次性的，有有效期，需要尽快传给后端。

---

### 二、后端换取用户标识

前端将 code 发送给后端，后端调用微信接口：

* 换取 `openid`
* 获取 `session_key`

`openid` 用于唯一标识用户。

---

### 三、生成登录态（token）

后端根据 openid：

* 查询用户是否已存在
* 不存在则创建用户
* 生成系统内的登录态（如 JWT token）

并将 token 返回给前端。

---

### 四、前端保存登录态

前端拿到 token 后，一般会：

* 存入本地（storage）
* 注入到请求头（Authorization）

后续接口请求都带上 token，实现鉴权。

---

### 五、用户信息获取（可选）

如果需要用户头像、昵称等信息：

* 通过 `wx.getUserProfile` 获取用户授权信息
* 再传给后端保存

---

### 六、登录态校验与续期

在实际项目中，还需要处理：

* token 过期（401 处理）
* 自动重新登录（重新调用 wx.login）

保证用户体验流畅。

---

### 七、总结

小程序登录流程可以总结为：

> **前端获取 code → 后端换 openid → 生成 token → 前端保存并用于后续请求。**

核心点在于：

* code 一次性且短期有效
* 登录态由后端维护
* 前端只负责传递和使用 token
