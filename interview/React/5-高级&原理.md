1. 什么是虚拟 DOM？React 的 diff 算法大致原理是什么？

   虚拟 DOM（Virtual DOM）是一个轻量级的 JavaScript 对象，用来描述真实 DOM 的结构。

   ```js
   // JSX 代码
   <div className="container">
    <h1>Hello</h1>
    <p>World</p>
   </div>

   // 对应的虚拟 DOM（简化版）
   {
    type: 'div',
    props: { className: 'container' },
    children: [
        { type: 'h1', props: {}, children: ['Hello'] },
        { type: 'p', props: {}, children: ['World'] }
    ]
   }
   ```

   工作流程：`状态变化 → 新虚拟 DOM → diff 对比旧虚拟 DOM → 计算差异 → 批量更新真实 DOM`

   **React diff 算法核心原理**
   - 策略一：不同类型的元素直接销毁重建

     ```js
     <!-- 旧虚拟 DOM -->
     <div>...</div>

     <!-- 新虚拟 DOM -->
     <span>...</span>

     <!-- 结果：直接卸载 div，重新创建 span -->
     ```

   - 策略二：同一类型的 DOM 元素不跨层级移动

     ```js
     <!-- 旧虚拟 DOM -->
     <div className="old" title="tip">

     <!-- 新虚拟 DOM -->
     <div className="new" title="tip">

     <!-- 结果：保留 DOM 节点，只更新 className 属性 -->
     //保留同一 DOM 节点; 遍历新旧 props，更新变化的属性; 删除旧的多余属性
     ```

   - 策略三：同一类型的组件元素

     ```js
     // 旧组件
     <MyComponent name="old" />

     // 新组件
     <MyComponent name="new" />

     // 结果：组件实例不变，更新 props，触发生命周期（componentWillReceiveProps、componentDidUpdate）
     ```

   - 策略四：子节点的递归对比（最复杂）

     ```text
             开始对比
                 ↓
         新旧根节点类型是否相同？
             ↙        ↘
         相同        不同
             ↓          ↓
     保留 DOM    销毁旧树
     更新属性    重建新树
             ↓
     递归对比子节点
             ↓
     子节点是否有 key？
         ↙        ↘
     有 key     无 key
         ↓          ↓
     基于 key   按索引位置
     复用节点   依次对比
     ```
