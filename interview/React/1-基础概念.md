1.  什么是 React？它主要解决了什么问题？

    React是一个用于构建用户界面的JavaScript库。它由Facebook开发并开源，核心特点是以组件化和声明式的方式构建UI。

    主要解决的问题：
    - DOM操作复杂、性能差的问题。通过虚拟DOM和Diff算法，React可以批量更新、最小化DOM操作，提升性能。
    - UI状态难以管理和追踪的问题。
    - 代码复用困难的问题
    - 大型应用难以维护和扩展的问题

2.  JSX 是什么？使用 JSX 有什么好处？

    JSX是React提供的语法扩展，本质是React.createElement(component, props, ...children)函数的语法糖。

3.  React 中的 state 和 props 有什么区别？
    - props是单向数据流，不能再子组件修改父组件传过来的props，要在父组件修改。
    - state是在组件内部管理，可以修改。
    - props和state的改变都会导致组件重新渲染。

4.  为什么在 React 中要使用 `key` 属性？如果不使用会怎样？

    key是React用来识别列表中每个元素的唯一标识符，帮助React在更新虚拟DOM时高效判断哪些元素发生了变化、被添加或被删除。

    不适用key，会导致：
    - 性能下降。React无法复用组件，需要重新创建
    - 状态错乱。组件内部state可能会意外保留或丢失
    - 渲染错误。列表顺序变化时，UI可能会出现意料之外的结果

5.  React 的核心思想是什么？它和传统 MVC 有什么本质区别？

    React 的核心思想可以概括为三个关键词：**声明式 UI**、**组件化**、**单向数据流**。
    - **声明式 UI（Declarative UI）**

      你描述 UI 在任意状态下应该长什么样，React 负责实现它。

    - **组件化（Component-Based）**

      将 UI 拆分成独立、可复用的组件，每个组件管理自己的状态和逻辑。

    - 3. **单向数据流（One-way Data Flow）**

      数据从父组件流向子组件（通过 props），变化只能通过特定的方式向上传递。

    MVC 是**命令式**、**多向流动**的架构，需要手动维护 UI 与数据的同步；React 是**声明式**、**单向流动**的库，状态变化驱动 UI 自动更新，极大地简化了复杂 UI 的开发。
