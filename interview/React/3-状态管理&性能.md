1. setState 是同步还是异步的？请说明原因。

   setState 在绝大多数场景下表现为“异步”的，但其实质是“批处理（Batching）”。

   现象
   - 表现上是异步：当调用setState后立刻去访问state，拿到的通常是旧值。
   - 实质上是同步：setState本身的执行过程是同步的，只是它触发的状态更新和组件渲染时被React调度的，并不是立刻执行。

   核心原因
   - 如果setState是同步的，由于props只有在父组件重绘时才会更新，这会导致在同一个方法中state变了但props没变，破坏了数据一致性。
   - 性能优化：如果在一个事件处理函数中调用了10次setState，React只会触发1次重新渲染。如果是同步的，浏览器会经历10次重绘，极其浪费资源。
   - 提升用户体验：React可以根据任务优先级（如动画vs数据流）来推迟某些非紧急的渲染，确保页面操作流程。

   如何在更新后拿到最新值
   - 使用函数式更新：setCount(prevCount => prevCount + 1);
   - 使用useEffect监听状态的变化

   在 React 18 及其后续版本中，`setState` 始终表现为**异步批处理**，目的是为了性能优化和保持内部一致性。它通过将多个更新请求放入队列，等到当前执行流结束再统一触发一次渲染。如果需要获取最新状态，推荐使用 `useEffect` 或者 `setState` 的函数式参数写法。

2. React 中如何避免不必要的重新渲染？

   React的渲染机制是：当父组件渲染时，默认情况下其所有子组件都会递归地重新渲染。要打破这个默认行为，我们可以从组件跳过、属性缓存和状态管理三个维度上手。
   - 组件缓存：React.memo

     React.memo是一个高阶组件，它会对组件的props进行浅比较。如果props没有变化，react将会跳过对该组件的渲染。

   - 引用类型的稳定化：useMemo与useCallback

     即使使用了 React.memo，如果父组件在渲染时重新创建了对象、数组或函数，子组件依然会触发渲染（因为浅比较会发现引用的地址变了）。

     useCallback用于缓存函数引用，防止每次渲染都生成新的匿名函数。

     useMemo用于缓存计算结果或数组/对象，避免昂贵的重复计算。

   - 状态下移

     如果一个状态只在某个子组件中使用，不要把它放在父组件里。

   - 使用组件组合

     利用 children 或将组件作为 props 传递，可以有效避开渲染路径。

     ```js
     function Layout({ children }) {
       const [count, setCount] = useState(0);
       return (
         <div onClick={() => setCount((c) => c + 1)}>
           {/* 即使 Layout 渲染，children 指向的内容不会因为 count 改变而重新生成 */}
           {children}
         </div>
       );
     }
     ```

3. React.memo、useMemo、useCallback 分别有什么作用？它们的区别是什么？

React.memo（高阶组件）

```js
const Child = React.memo(({ name, onClick }) => {
  console.log("Child rendered");
  return <button onClick={onClick}>{name}</button>;
});

// 只有 props 变化时才重新渲染
// 注意：如果 onClick 每次都是新函数，memo 会失效
```

useMemo（缓存值）

```js
function Component({ list, filter }) {
  // 仅当 list 或 filter 变化时重新计算
  const filteredList = useMemo(() => {
    return list.filter((item) => item.includes(filter));
  }, [list, filter]);

  return <List data={filteredList} />;
}
```

useCallback（缓存函数）

```js
function Parent() {
  const [count, setCount] = useState(0);

  // 每次 count 变化，handleClick 仍然是同一个函数引用
  const handleClick = useCallback(() => {
    console.log("clicked");
  }, []); // 空依赖，永远不变

  return <MemoChild onClick={handleClick} />;
}
```

**作用**

| API               | 作用                          | 缓存的是什么                             | 主要使用场景                                   |
| ----------------- | ----------------------------- | ---------------------------------------- | ---------------------------------------------- |
| **`React.memo`**  | 防止组件 props 未变时重新渲染 | 整个组件的渲染结果                       | 纯展示组件、频繁被父组件重渲染的子组件         |
| **`useMemo`**     | 缓存**计算过程的结果**（值）  | 任何类型的返回值（对象、数组、原始值等） | 复杂计算、派生数据、保持对象引用稳定           |
| **`useCallback`** | 缓存**函数本身**              | 函数定义                                 | 传递给子组件的回调函数，配合 `React.memo` 使用 |

**区别**

|维度|React.memo|useMemo|useCallback|
|---|---|---|---|
|**本质**|组件优化包装器|Hook|Hook|
|**返回类型**|新的组件|缓存的值|缓存的函数|
|**谁在使用**|组件定义时包裹|函数组件内部调用|函数组件内部调用|
|**依赖项**|自动浅比较 props|手动指定依赖数组|手动指定依赖数组|
