
# 面试题：如何避免组件重复渲染？（标准回答）

在 React 中避免组件重复渲染，核心思路是：**减少不必要的状态变化 + 控制引用稳定性 + 做渲染边界隔离**。

React.memo、useMemo（缓存计算值）、useCallback（缓存函数）、合理拆分组件。

## 一、组件层优化

* 使用 **React.memo** 包裹纯展示组件，基于 props 浅比较，避免父组件更新导致的无效重渲染。
* 合理拆分组件，把高频变化的状态下沉到更小的子组件中，减少大组件整体刷新。

## 二、引用稳定性

* 使用 **useCallback** 缓存函数，避免每次 render 都创建新函数，导致子组件重新渲染。
* 使用 **useMemo** 缓存计算结果或对象，避免每次生成新引用（如对象/数组）。

## 三、状态设计

* 避免把所有状态放在同一个组件里，进行**状态拆分**（按业务域或更新频率划分）。
* 减少不必要的 setState，合并更新（批量更新）。

## 四、列表与 key

* 合理使用 **key**，确保列表 diff 稳定，避免整列表重建。
* 大列表场景使用**分页加载/虚拟列表**，减少一次性渲染节点数量。

## 五、全局状态优化

* 在 Redux/MobX 中，按需订阅（selector/observer），避免整个组件树因全局状态变化而刷新。
* 对于高频更新数据（如购物车数量），尽量做**局部订阅**或拆分 store。

## 六、实际项目中的做法

在零售小程序中，我们对商品列表、购物车模块做了这些优化：

* 商品卡片组件使用 React.memo，避免列表滚动时重复渲染
* useCallback 缓存加购函数，防止子组件重复更新
* 购物车使用局部状态 + 全局状态结合，只更新变更项（增量更新）
* 列表采用分页加载，避免一次性渲染大量商品

## 七、总结

避免重复渲染的核心是：

> **控制状态变化范围 + 保持引用稳定 + 做好组件边界划分**

从这三个维度入手，基本可以解决大部分性能问题。
