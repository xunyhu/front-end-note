1.  什么是受控组件和非受控组件？各有什么适用场景？

    表单元素由两种数据管理方式分为受控和非受控，受控组件的状态由state控制，非受控组件由DOM自身控制、通过ref读取；受控组件可以边输入边验证，非受控组件需要提交时才能验证；受控组件可以动态设置值，非受控组件不能动态设置值；

    受控组件适用场景：实时表单验证、动态控制输入、条件禁用启用、联动选择（省市选择）
    非受控组件适用场景：简单表单（不需要实时验证）、文件上传（必须）、大型表单（不需要实时验证，避免重复渲染）
    实际项目中通常会混合使用，需要实时验证的字段用受控，复杂字段用非受控（如富文本编辑器），文件上传用非受控。

2.  React 的生命周期方法（16.3 之前与之后）有哪些？函数组件中如何模拟生命周期？

    16.3之前，生命周期分为三个阶段：**挂载（Mounting）**、**更新（Updating）**、**卸载（Unmounting）**。

    | Class 组件方法                   | Hooks 替代方案                  | 执行时机   |
    | -------------------------------- | ------------------------------- | ---------- |
    | `constructor`                    | `useState` + 初始化函数         | 组件创建时 |
    | `componentDidMount`              | `useEffect(() => {}, [])`       | 首次渲染后 |
    | `componentDidUpdate`             | `useEffect(() => {})`（无依赖） | 每次渲染后 |
    | `componentDidUpdate`（条件监听） | `useEffect(() => {}, [deps])`   | 依赖变化时 |
    | `componentWillUnmount`           | `useEffect` 的清理函数          | 组件卸载前 |
    | `getDerivedStateFromProps`       | `useEffect` 监听 props 变化     | 渲染过程中 |
    | `shouldComponentUpdate`          | `React.memo` / `useMemo`        | 重新渲染前 |
    | `getSnapshotBeforeUpdate`        | `useLayoutEffect` + ref         | DOM 更新前 |
    | `componentDidCatch`              | 暂无 Hooks 版本                 | 错误捕获时 |

3.  什么是高阶组件（HOC）？它与 Render Props、Hooks 相比有什么优缺点？

    **高阶组件**是一个**函数**，它接收一个组件作为参数，并返回一个**增强后的新组件**。

    ```jsx
    // HOC 的基本结构
    const EnhancedComponent = higherOrderComponent(WrappedComponent);

    // 示例：给组件添加日志功能
    function withLogging(WrappedComponent) {
      return class extends React.Component {
        componentDidMount() {
          console.log(`组件 ${WrappedComponent.name} 已挂载`);
        }

        render() {
          return <WrappedComponent {...this.props} />;
        }
      };
    }

    // 使用 HOC
    const EnhancedButton = withLogging(Button);
    <EnhancedButton onClick={handleClick} />;
    ```

    HOC 的常见应用场景: 权限控制（条件渲染）、性能监控、样式注入、数据获取和注入。

    **Render Props** 是一个组件属性，其值是一个函数，该函数返回一个 React 元素。组件通过调用这个函数来决定渲染什么。

    **Hooks** 是 React 16.8 引入的函数式 API，允许在函数组件中使用 state 和其他 React 特性。

    HOC 通过组件包装增强功能，Render Props 通过函数传递实现复用，而 Hooks 以最自然的方式将状态逻辑抽离，是当前 React 开发的首选方案。

4.  组件生命周期与 Hooks

    | 阶段                | Class 组件                 | 函数组件 Hooks            | 执行时机     | 典型用途                 |
    | ------------------- | -------------------------- | ------------------------- | ------------ | ------------------------ |
    | **挂载前**          | `constructor`              | `useState` 初始化         | 组件创建时   | 初始化 state、绑定 this  |
    | **挂载后**          | `componentDidMount`        | `useEffect(() => {}, [])` | DOM 插入后   | API 请求、DOM 操作、订阅 |
    | **更新前**          | `getDerivedStateFromProps` | `useEffect` + 依赖        | props 变化时 | 根据 props 更新 state    |
    | **更新前(性能)**    | `shouldComponentUpdate`    | `React.memo`              | 重渲染前     | 性能优化，跳过渲染       |
    | **更新前(读取DOM)** | `getSnapshotBeforeUpdate`  | `useLayoutEffect`         | DOM 更新前   | 读取滚动位置等           |
    | **更新后**          | `componentDidUpdate`       | `useEffect`               | 每次渲染后   | 响应 props/state 变化    |
    | **卸载前**          | `componentWillUnmount`     | `useEffect` 清理函数      | 组件移除前   | 清理订阅、定时器         |
    | **渲染**            | `render`                   | 函数体本身                | 每次渲染     | 返回 JSX                 |
    | **错误捕获**        | `componentDidCatch`        | 暂无 Hooks 版本           | 渲染错误时   | 错误边界                 |
