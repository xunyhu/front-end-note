## 首屏加载慢的原因

- 资源过大（JS / 图片 / CSS）
- 请求过多
- 渲染阻塞（CSS / JS）
- 接口慢
- 白屏时间长
  区分 **网络层 / 渲染层 / 业务层**

## 常见首屏优化手段有哪些

- 资源优化
  - 代码拆分（code splitting）
  - Tree Shaking
  - gzip / brotli
- 渲染优化
  - SSR / 预渲染
  - 骨架屏
- 请求优化
  - 合并请求
  - 缓存
- 加载策略
  - 懒加载
  - 预加载（preload / prefetch）

## 如何减少白屏时间

- inline critical CSS
- script defer
- SSR
- 资源预加载

## 图片很多的页面怎么优化首屏

- 懒加载（IntersectionObserver）
- WebP / AVIF
- CDN
- 尺寸裁剪
- LQIP（低质量占位图）

## Tree Shaking 和 Code Splitting 有什么区别

- Tree Shaking：删除无用代码
- Code Splitting：拆包按需加载

## 如果让你优化一个电商首页首屏，你怎么做

**1️⃣ 先分析问题**

- Performance 面板
- Lighthouse
- 看 LCP / FCP

**2️⃣ 资源层**

- 拆包（React.lazy）
- CDN + gzip

**3️⃣ 渲染层**

- SSR 或预渲染
- 骨架屏

**4️⃣ 数据层**

- 接口并发
- 缓存（localStorage / SWR）

**5️⃣ 细节优化**

- 图片懒加载
- 字体优化
- DNS 预解析

## 小程序首屏优化怎么做

- 减少 setData 次数
- 分包加载
- 首屏数据提前返回
- 骨架屏
- 组件按需渲染

## 如何量化优化效果

- Lighthouse
- Web Vitals
- Performance API
- 埋点监控（自研 or Sentry）
